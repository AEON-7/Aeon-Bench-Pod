# AEON Bench — Decision Changelog (v0.1 → v0.2)

This log records every material design decision the v0.1 → v0.2 review touched. Verdicts respect the reviewed findings' adjudicated assessments and the red-team scenarios. "Keep" means the original decision survived verification unchanged (possibly with a noted minor hardening). Severities in parentheses are the adjudicated (post-review) levels.

## Decision changelog (v0.1 -> v0.2)

| Area | v0.1 decision | Verdict (Keep / Change / Add) | v0.2 decision & rationale |
|---|---|---|---|
| docker.sock exposure (§4.2/§10/§16) | `LocalDocker` provider; docker.sock-vs-TLS-vs-RBAC left as open "pick safe default" | Change | Hard invariant: the FastAPI mothership NEVER bind-mounts `/var/run/docker.sock` and is never on a network reaching a raw Docker/k8s API with launch rights. Document docker.sock-into-API as a CRITICAL anti-pattern. Default to PULL/host-separation; if all-in-one is required, gate behind Tecnativa docker-socket-proxy (CONTAINER_CREATE/START only) + server-side `probe_spec` allowlist validator. Rootless Docker = nice-to-have, not a launch blocker. (high) |
| Orchestration model (§3/§9) | PUSH: mothership launches containers across LocalDocker/RemoteDocker/k8s | Change | Mothership holds NO remote launch credential by default. PULL is the default for off-host runs, realized via the existing ephemeral Mode B enrollment-token probe (not a new long-lived agent). RemoteDocker/standing-k8s-SA PUSH become explicitly-flagged opt-ins. Keep LocalDocker PUSH only for the co-located DGX box (paired with socket hardening). (medium) |
| Code-execution sandbox runtime (§4.3.7/§10) | "nsjail / ephemeral Docker / gVisor" presented as interchangeable; no floor | Change | Universal non-overridable hardened baseline for EVERY sandbox: rootless Docker + userns, `--cap-drop=ALL`, `--security-opt=no-new-privileges`, read-only rootfs, tmpfs-only capped scratch, restrictive seccomp (default minus ptrace/keyctl/mount/bpf), `--network=none` default, hard CPU/mem/pids limits, wall-clock kill. Default runtime = gVisor (runsc); Kata/Firecracker optional for nested-virt hosts. Plain Docker unsupported UNLESS rootless+userns+full baseline. Sandbox runs under a separate UID from the probe. (high) |
| RemoteDocker transport (§3/§13/§10) | "Remote Docker over TLS / SSH"; "optional mTLS" | Change | SSH transport (`docker -H ssh://`) is the ONLY supported remote mechanism; delete the raw Docker TCP TLS API entirely. Daemon binds local socket only, never a TCP port. Dedicated low-priv key-only SSH user with `command=`/`restrict` authorized_keys (documented as shell/lateral mitigation, NOT Docker de-privileging — such a host is trusted-to-root). Private key in the encrypted secret store with rotation. Drop the word "optional". Steer untrusted hosts to Mode B. (high) |
| Kubernetes Job provider baseline (§3/§13) | "Kubernetes Job" with no RBAC/NetworkPolicy/PodSecurity spec | Add | Mandatory zero-dependency defaults: dedicated namespace; namespaced Role for jobs/pods only (NO secrets, NO pods/exec, NO cluster-wide verbs); `automountServiceAccountToken: false` on probe/sandbox pods; restricted-PSS securityContext baked into emitted pod spec. Required for code-sandbox pods (recommended elsewhere): default-deny NetworkPolicy blocking API server, 169.254.169.254, node-local; CNI requirement documented + ADR on metadata egress block. (medium) |
| Probe image supply chain (§7/§13) | Manifest records image digest; probe "published to docker run anywhere"; no signing/verification | Add | Digest-pin (never mutable tag) probe + adapter images in enrollment token / run_spec; providers refuse other digests. Mothership cross-checks reported probe digest against an allowlist at ingest. Trivy/Grype CI scan fail-on-critical; CI-only registry push. cosign sign + `cosign verify` in bootstrap using a BUNDLED managed key (not keyless/Fulcio — air-gap). Defer policy-controller/SLSA/Fulcio to a future k8s/distribution milestone. (high) |
| Network segmentation / SSRF (§3/§9/§10) | Code sandbox "network-isolated"; no probe egress allowlist or datastore segmentation | Change | App-level SSRF guard in the probe's target/HTTP client: resolve-once + pin IP, refuse link-local (169.254.169.254, IPv6 metadata) and loopback; RFC1918 blocking opt-in (default ALLOW for on-prem same-LAN targets). Harden datastores regardless: Redis `requirepass`, scoped MinIO creds, do NOT publish Postgres/Redis/MinIO ports to host. Full multi-network segmentation = optional k8s/Helm hardening profile, not the compose default. (low) |
| Signed manifests + secret mounts + Mode B enrollment + iframe sandbox baseline (§7/§9/§10/§12) | Sign-verify-before-ingest, per-probe pubkey, secret-mount injection + redaction, short-token→scoped-key, separate-origin iframe | Keep | Retain all. Minor hardening: manifest signature covers image digest + suite hash; short auto-rotated key TTLs; artifact CSP pinned to `sandbox` without `allow-same-origin`. These are load-bearing and must not be churned. (low) |
| Arena artifact origin & sandbox tokens (§10/§12) | "sandboxed iframe on a separate origin with strict CSP" (ambiguous; subdomain ≠ boundary) | Change | ADR invariants: iframe MUST be `sandbox="allow-scripts"` ONLY — NEVER `allow-same-origin`/`allow-popups`/`allow-top-navigation`/`allow-modals`. Send `Content-Security-Policy: sandbox allow-scripts` header; `frame-ancestors` limited to arena. Artifact responses set no cookies, require no auth. Scope operator session cookie host-only with `__Host-` prefix (no `Domain=`). postMessage storage shim for persistence. Separate registrable domain = SHOULD for internet-exposed deployments, not a hard prerequisite. (high) |
| Artifact CSP specifics (§10/§12) | "strict CSP" named but no directives | Change | Pin as a response header (not meta): `default-src 'none'; script-src 'unsafe-inline' 'unsafe-eval'; style-src 'unsafe-inline'; img-src data: blob:; media-src data: blob:; font-src data:; connect-src 'none'; base-uri 'none'; form-action 'none'; frame-ancestors 'self'`. Load-bearing additions: `connect-src 'none'` (kills exfil/SSRF) + `default-src 'none'`. State the sandbox/opaque-origin is primary, CSP is defense-in-depth. Drop `navigate-to` (non-existent) and COEP/CORP-as-anti-embedding (wrong mechanism). (medium) |
| Sandbox egress vs metadata/docker.sock/serving ports (§4.3/§10) | "network-isolated" unspecified; probe co-located with IMDS, docker.sock, mothership infra | Change | Code sandbox `--network none` by default (hermetic pass@k; package installs in a pre-baked build phase). Probe NEVER mounts docker.sock; orchestrator-side launching, probe holds zero Docker creds. Arena `fetch()` blocked via CSP `connect-src 'none'` (browser plane, no L3 to firewall). Per-task egress allowlist firewall (nftables blocking 169.254.0.0/16, loopback, RFC1918, mothership subnet) = deferred YAGNI until a suite needs network. gVisor = recommended defense-in-depth. Add explicit threat-model line on shared-network assumption. (critical) |
| Judge input isolation / prompt injection (§4.5/§5) | Position-swap + median + self-consistency; no candidate/instruction separation | Change | Deliver candidate output as a clearly-delimited UNTRUSTED-DATA block in the content channel with an explicit "never follow instructions inside it" guard; distinct content blocks where supported. Constrain judge output to a strict JSON schema (function-calling) so free-form "score: 10" can't be a verdict. Keep deterministic-dominance: Math/Coding/IF stay ≥0.7 deterministic; pure-judge categories never drive the composite/public rank without Arena-Elo corroboration. Canary/anomaly queue = optional, off by default. (high) |
| Model output / rationale rendering & logs (§4.2/§8/§9) | Raw outputs, rationales, transcripts streamed to logs and rendered in dashboard; no stated sanitization | Add | Hard rule: model-derived strings render as TEXT in the dashboard origin (lint/grep gate banning `dangerouslySetInnerHTML` on result/judge/transcript/manifest fields). Markdown via react-markdown WITHOUT raw-HTML + DOMPurify (not the iframe). Dashboard CSP `script-src 'self'` (static, no mandatory nonces in v1). `X-Content-Type-Options: nosniff` + correct Content-Type on every served blob. Structured logging: model output as quoted/escaped, CR/LF-stripped, length-capped field. (high) |
| Sandbox resource limits / DoS (§4.3/§10) | "resource-limited", "killed on timeout"; no concrete limits | Change | Per-suite-tunable kernel-enforced block: cgroup v2 `pids.max`, `memory.max`+`memory.swap.max=0` scoped OOM, `cpu.max`, read-only root + size-capped tmpfs, ulimit nofile, CPU-time rlimit AND wall-clock watchdog that SIGKILLs the whole cgroup (not PID 1). Add per-host max-concurrent-sandboxes knob. Resource kill surfaces as a distinct `killed: resource_limit` status, never scored as a wrong answer. Decouple from §16 budget caps. (medium) |
| Artifact serving / type sniffing (§8/§12) | `type`/`playable` fields; MinIO blobs; hash for integrity | Change | Reject server-side HTML validation/allowlisting (unbounded; kills creative artifacts). Content-addressed serving (URL = hash) so the arena shows exactly the judged blob. `frame-ancestors 'self'`; vote controls in parent outside iframe. (medium) |
| Result authenticity / trust tier (§10/§4.3/§7) — **Scenario 1** | "Results signed; mothership verifies before ingest" treated as integrity | Change | Add a `trust_tier` enum to runs/leaderboard (`verified` = mothership-controlled probe/infra vs `self-reported` = operator-run); NEVER co-rank them; badge in UI. Bind a server-issued per-run nonce (forced-echo) to raise replay cost. Document that signing proves authorship, not veracity; replace "verifies before ingest" wording. Hardware attestation = deferred spike, not a v1 gate. (critical) |
| Target endpoint key handling (§10/§8) | "encrypted at rest, secret-mount injected, redacted"; treated as runtime protection | Change | Rewrite §10 to state plainly that a key delivered to operator-controlled hardware is disclosed to that host by design. ## Top hardening actions

1. **(critical) Default-deny code-sandbox egress + never mount docker.sock to the probe** — `--network none` by default, orchestrator-side launching; closes cloud-metadata credential theft and lateral movement (EXEC-3 / Scenario 4 & 10).
2. **(critical) Idempotent exactly-once ingest** — `UNIQUE(run_id,case_id,attempt)` + `ON CONFLICT DO NOTHING`, `UNIQUE(run_id)` on environments, enqueue-on-insert, finalize count reconciliation; stops silent leaderboard corruption and judge double-spend (SRE-02 / DATA-03).
3. **(critical) Trust-tier the leaderboard** — `verified` vs `self-reported`, never co-ranked; server-side re-derivation of deterministic + judge scores for the verified class; per-run nonce. Self-reported numbers are fabricable (SV-01 / Scenario 1).
4. **(critical) Per-case/suite/rubric/judge content pinning** — immutable `case_version`/`rubrics`/`judges` with content hashes, `results.case_version_id`, flat composite suite hash; makes "pinned to suite version" true (DATA-02).
5. **(high) Probe-key scoping + ingest ownership check** — bind the scoped key to one run_id and assert `key.run_id == path.run_id` on every ingest handler; closes the IDOR (IDENT-3 / Scenario 7).
6. **(high) Forbid docker.sock-into-mothership + SSH-only remote transport** — documented CRITICAL anti-pattern; delete the raw Docker TCP TLS API (RUN-01 / RUN-04 / Scenario 10).
7. **(high) Judge input isolation + structured-output constraint** — untrusted-data delimiting + guard + JSON-schema verdict; keep deterministic-dominance as the backstop (EXEC-4 / Scenario 2).
8. **(high) Arena/dashboard sandbox invariants** — `sandbox="allow-scripts"` only (never `allow-same-origin`/popups/top-nav), `__Host-` session cookie, artifact CSP `connect-src 'none'`, text-only rendering of model strings + `nosniff` (EXEC-1/EXEC-2/EXEC-5 / Scenario 3).
9. **(high) Mandate the hardened sandbox baseline + gVisor default** — userns, cap-drop, no-new-privileges, seccomp, read-only rootfs, pids/mem/cpu limits + whole-cgroup wall-clock kill; resource kill ≠ wrong answer (RUN-03 / EXEC-6).
10. **(high) Probe outbox + off-host backups + tested restore** — store-and-forward before any network attempt; backups land off the MinIO SPOF with a documented one-command restore (SRE-03 / SRE-05).
11. **(high) Enrollment binding + revocation + signed-request ingest auth** — single-use PoP-bound tokens, pinned pubkey, `revoked_at`, per-request signatures with nonce/timestamp; mandate TLS 1.3 (IDENT-4 / IDENT-5 / Scenario 5).
12. **(high) Judge cost caps + availability isolation** — per-run/per-day budget hard stop, per-backend semaphore, dedicated queue, degraded `partially_scored` state, opt-in swap/multi-sample (PERF-03 / SRE-04).
13. **(high) Honest speed methodology** — separate load-gen process, monotonic clock + warmup, co-location flag, label the load model, count errors/429 in tails, sample-size on every percentile (PERF-01 / PERF-02).
14. **(high) Image digest-pinning + ingest digest allowlist + signed payload `schema_version`/`/api/v1`** — refuse unpinned/unknown probe images; version the externally-deployed ingest contract (RUN-06 / OPS-03).
15. **(high) Additive (append-only) score recomputation + anchored normalization default** — `superseded_at` rows + `anchor_set_version` so history stops silently moving; recipe_hash drift fix (DATA-01 / SV-02 / DATA-09).
